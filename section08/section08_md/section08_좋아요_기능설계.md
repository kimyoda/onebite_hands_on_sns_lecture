## section08*좋아요기능*설계구조

- 좋아요 기능은 대부분의 SNS가 제공하는 기능과 똑같다

- 아이콘 채우기 -> 좋아요 눌렀는지 체크, 좋아요 카운트 증가

포스트 테이블에 SELECT(post)를 한다

Post 테이블에 저장된 LikeCount = 0 값을 1 증가시키는 UPDATE(post)를 작동시켜 실제로 1을 증가시키면 된다

likeCount = likeCount + 1

좋아요 기록을 보관하는 Like 테이블을 추가하고

INSERT(like)를 추가하여 user_id: aaa, post_id:3 like 테이블을 통하여 누가 좋아요를 눌렀는 지 추가하는 기능을 추가한다.

두 명의 유저가 똑같은 시간에 동시에 누를 경우 제대로 업데이트 되지 않을 수도 있다.

두 유저에게 모두 각각 likeCount = 0 이라는 응답을 보내고 각각 0 + 1 응답을 보낼 때 동시에 0을 보냈기에 동일한 값인 1이라는 업데이트 식의 요청을 보내며, 좋아요 버튼은 두번 눌린게 되지만, 1로만 업데이트 시켜

post_table에 1로 업데이트 되는 버그가 발생할 수 있다

위와 같은 이슈를 **동시성 이슈**라고 한다
-> 하나의 데이터에 여러 사용자가 접근할 때 발생할 수 있는 문제

- 은행의 송금 시스템
- 쇼핑몰의 재고 관리 시스템
- 식당 및 장소 예약 관리 시스템 등

-> 이를 해결하기 위해
추가 옵선 즉 업데이트 전까지 행을 잠그는 SELECT (post)를 보낼때 설정한다 (SQL 문을 통해)
SLECT(post)를 한번 더 보내도 대기를 한 뒤에,
SELECT의 결과로 likceCount = 0을 보내고 이후 UPDATE(post)를 통해 업데이트 요청을 보낸다.
그와 동시에 업데이트가 되어 잠금까지 풀려 대기중이던 처음 보냈던 유저의 요청이 likeCount = 1로 증가되어 응답되고
최종으로 UPDATE(post)되어 likceCount = 1 + 1으로 응답받을 예정으로

동시성 이슈를 쉽게 해결할 수 있다.

Supabase Client는 행을 잠그는 등의 디테일한 옵션을 제공되지 않는다.

Supabase에 제공되는 PRC가 제공된다(RemoteProcedure Call)

- 원격으로 함수를 호출하는 기능
- 데이터베이스 전용 함수를 만들어 저장 후 ReactApp에서 원격 호출을 하는 기능이다.
