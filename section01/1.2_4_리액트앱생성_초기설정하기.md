```bash

```

### tsconfig 파일 간략 설명

- **`tsconfig.app.json`**
  - React 앱 소스(`src`)를 위한 TypeScript 설정 파일이다.
  - 브라우저 환경 기준으로 JSX, 모듈 해석, 타입 체크 규칙 등을 담당한다.

- **`tsconfig.node.json`**
  - Node 환경에서 실행되는 파일용 설정한다.
  - 예: `vite.config.ts` 같은 설정 파일의 타입 체크에 사용된다.

- **`tsconfig.json`**
  - 루트(기준) 설정 파일입니다.
  - `tsconfig.app.json`, `tsconfig.node.json`을 참조(`references`)해 프로젝트 전체 타입 체크 구조를 관리한다.

### ESLint 설정 추가 내용과 이유

`eslint.config.js` 파일에 아래 규칙을 추가한다.

```js
"@typescript-eslint/no-unused-vars": "off",
"@typescript-eslint/no-explicit-any": "off",
추가 이유:

@typescript-eslint/no-unused-vars를 끈 이유
실습 과정에서 임시 변수나 아직 사용하지 않은 변수가 자주 벌생한다.
불필요한 경고를 줄이고 학습 흐름을 유지하기 위함이다.

@typescript-eslint/no-explicit-any를 끈 이유
초기 구현 단계에서 any 타입을 유연하게 허용해 기능 구현 속도와 이해도에 집중하기 위함이다.

TypeScript 옵션 주석 처리 내용과 이유
tsconfig.app.json(또는 해당 설정 파일)에서 아래 옵션을 주석 처리한다.

// "noUnusedLocals": true,
// "noUnusedParameters": true,
주석 처리 이유:

실습 중에는 아직 사용하지 않는 로컬 변수나 파라미터가 자주 생성된다.
이 옵션을 켜두면 불필요한 오류/경고로 진행이 끊길 수 있으므로, 강의 초반에는 주석 처리하여 기능 구현 흐름을 우선한다.
```
