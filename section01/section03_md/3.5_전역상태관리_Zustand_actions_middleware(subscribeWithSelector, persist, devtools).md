## 3.4 Zustand_middleware(comebine, immer)

- Zustand는 미들웨어를 통해 store에 다양한 기능을 추가할 수 있다.
- 대표적인 5가지가 있다.
  | 미들웨어 | 한 줄 요약 |
  | :--- | :--- |
  | **combine** | 상태와 액션을 분리 정의 + 타입 자동 추론 |
  | **immer** | 불변성 걱정 없이 상태를 직접 수정 가능 |
  | **subscribeWithSelector** | 특정 상태 변화만 감지해서 구독 |
  | **persist** | 상태를 `localStorage` 등에 자동 저장 |
  | **devtools** | Redux DevTools로 상태 변화 디버깅 |

### 1. combine - 상태와 액션을 분리

- `combine`은 store를 만들 때 **첫 번째 객체엔 상태, 두 번째 객체엔 액션**을 분리해서 정의할 수 있게 해준다.
- 장점은 **타입을 별도로 지정하지 않아도** 첫 번째 객체의 상태 타입이 자동으로 추론된다는 것이다.

```ts
import { create } from "zustand";
import { combine } from "zustand/middleware";

export const useCountStore = create(
  combine(
    // 첫 번째 객체 - 상태 (타입 자동 추론)
    { count: 0 },
    // 두 번째 객체 - 액션
    (set) => ({
      actions: {
        increaseOne: () => set((state) => ({ count: state.count + 1 })),
        decreaseOne: () => set((state) => ({ count: state.count - 1 })),
      },
    }),
  ),
);
```

- **타입 자동 추론은 첫 번째 객체(상태)에만 적용**된다.
- 두 번째 객체의 액션 타입은 추론되지 않으니, 액션 쪽에 복잡한 타입이 필요하면 직접 지정해야 한다.

### 2. immer - 불변성 걱정 없이 상태 직접 수정

- React에서 상태를 변경할 때 **불변성(Immutability)**을 지켜야 한다.
- 기존 객체를 직접 수정하는게 아니라 새 객체를 만들어 반환해야 한다.
- `immer`는 이 문제를 편하게 해준다.
- **내부적으로 불변성을 알아서 처리**하기 때문에 상태를 직접 수정하는 것처럼 편하게 쓸 수 있다.

```ts
import { immer } from "zustand/middleware/immer";

// immer가 combine을 감싸는 구조
export const useCountStore = create(
  immer(
    combine({ count: 0 }, (set, get) => ({
      actions: {
        increaseOne: () => {
          // 직접 수정하는 것처럼 작성 — immer가 불변성 처리
          set((state) => {
            state.count += 1;
          });
        },
        decreaseOne: () => {
          set((state) => {
            state.count -= 1;
          });
        },
      },
    })),
  ),
);
```

- 미들웨어를 감싸는 순서는 바깥에서 안으로 `immer -> combine`순서이다.
- immer가 combine전체를 감싸야 상태 변ㄱ녕 전반에 불변성 처리가 적용된다.
