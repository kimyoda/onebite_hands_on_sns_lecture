## 3.4 Zustand_middleware(subscribeWWithSelector, persist, devtools)

- Zustand는 미들웨어를 통해 store에 다양한 기능을 추가할 수 있다.
- 대표적인 5가지가 있다.
  | 미들웨어 | 한 줄 요약 |
  | :--- | :--- |
  | **combine** | 상태와 액션을 분리 정의 + 타입 자동 추론 |
  | **immer** | 불변성 걱정 없이 상태를 직접 수정 가능 |
  | **subscribeWithSelector** | 특정 상태 변화만 감지해서 구독 |
  | **persist** | 상태를 `localStorage` 등에 자동 저장 |
  | **devtools** | Redux DevTools로 상태 변화 디버깅 |

### 1. subscribeWithSelector - 특정 상태 변화만 감지하기

- `subscribeWithSelector` 는 **특정 상태값이 바뀔 떄만** 롤백을 실행할 수 있게 해주는 미들웨어다.
- React의 `useEffect` 와 비슷한 역할을 하고, 차이점은 **컴포넌트 밖에서도 동작**한다는 것이다.
- `subscribe`는 store의 어떤 값이 바뀌든 무조건 롤백이 실행된다.
- `subscribeWithSelector`를 쓰면 **selector로 원하는 값만 골라서 감지**할 수 있다.

```ts
// 기본 subscribe — store 전체 변화 감지, 어떤 값이 바뀌든 실행됨
useCountStore.subscribe((state) => {
  console.log("뭔가 바뀜");
});

// subscribeWithSelector — count가 바뀔 때만 실행
useCountStore.subscribe(
  (store) => store.count, // selector — 감지할 값 선택
  (count, prevCount) => {
    // listener — 변화 감지 시 실행할 콜백
    console.log(count, prevCount); // 현재값, 이전값
  },
);
```

- 에시코드

```ts
useCountStore.subscribe(
  (store) => store.count,
  (count, prevCount) => {
    console.log(count, prevCount);

    // store 전체 상태를 가져올 수도 있다
    const store = useCountStore.getState();

    // store 상태를 직접 업데이트 할수도 있다
    // useCountStore.setState((store) => ( {...}))
  },
);
```

- `getState()` 로 현재 store 전체를 읽고, `setState()` 로 업데이트까지 가능, 컴포넌트 외부에서 store를 제어하는 로직을 작성할 때 유용하다.

### 2. persist + createJSONStorage - 새로고침해도 상태 유지

- `persist`는 store의 상태를 **브라우저 스토리지(localStorage, sessionStorage 등)에 자동으로 저장**해주는 미들웨어다.
- 새로고침해도 상태가 사라지지 않는다.
- `reateJSONStorage` 는 어떤 스토리지를 사용할지 지정하는 헬퍼 함수다.
- 기본값은 `localStorage`, `seesionStorage`나 커스텀 스토리지로도 바꿀 수 있다.

- localStorage, sessionStorage
- localStorage
  - 유지기간: 직접 삭제 전까지 영구 보존
  - 용도: 다크모드, 언어 설정 등
- sessionStorage
  - 유지기간: 탭/브라우저 닫으면 삭제
  - 용도: 로그인 세션, 임시 데이터 등

* 예시 코드

```ts
persist(
  subscribeWithSelector(
    immer(
      combine(...))
    ),
    {
      name: "countStore", // 스토리지에 저장될 key 이름
      // partialize - 저장할 상태만 선택 (actions는 저장할 필요가 없음)
      pertialize: (store) => ({
        // count만 저장
        count: store.count,
      }),

      // createJSONStorage - sessionsStorage에 저장하도록 설정
      storage: createJSONStorage(() => sessionStorage),
    }
)
```

- partialize가 중요한 이유

* `persist`는 기본적으로 **store 전체를 저장**하려고 한다.
* actions같ㅇ른 함수는 JSON으로 직렬화가 안되기 때문에 저장하면 오류가 생긴다.
* `partialize`로 저장이 필요한 상태값만 골라내야 한다.

```ts
// partialize 없으면 — actions 함수까지 저장 시도 → 오류
persist(store, { name: "countStore" });

// partialize로 count만 저장
persist(store, {
  name: "countStore",
  partialize: (store) => ({ count: store.count }),
});
```

### 3. devtools - ReduxDevTools로 상태 디버깅

- `devtools`는 브라우저의 Redux 확장 프로그램과 연결해주는 미들웨어다.
- store의 상태 변화를 시각적으로 추적할 수 있다.
- Reudx DevTools에서 볼 수 있는 것들은, 상태가 언제 어떻게 바뀌었는지, 각 액션 이름과 변경된 값, 과서 상태로 되돌리기(time travel)기능이 있다.
- 예시

```ts
devtools(
  persist(subscribeWithSelector(immer(combine(...)))),  // 감쌀 store
  {
    name: "countStore",  // DevTools에서 표시될 store 이름
  }
)
```

- 브라우저 확장 프로그램만 설치하면 된다.

### 순서를 지켜야 하는 이유

```ts
create(
  devtools(       // 4번째 — 가장 바깥
    persist(      // 3번째
      subscribeWithSelector(  // 2번째
        immer(    // 1번째
          combine({ count: 0 }, (set, get) => ({ ... }))  // 핵심
        )
      )
    )
  )
)
```

미들웨어는 **안쪽에서 바깥쪽으로 순서대로 실행**됩니다. 즉 가장 먼저 실행되어야 하는 핵심 로직이 안쪽에, 그걸 감싸는 부가 기능이 바깥쪽으로 나가는 구조예요.

각 미들웨어가 이 순서여야 하는 이유를 살펴보면 아래와 같습니다.

**`combine`이 가장 안쪽인 이유** — store의 기본 구조(상태 + 액션)를 정의하는 핵심이기 때문에 가장 먼저 실행되어야 합니다.

**`immer`가 combine 바로 바깥인 이유** — `set` 함수가 불변성을 처리하려면 상태 정의 직후에 감싸야 합니다. 다른 미들웨어들이 `set`을 호출하기 전에 immer가 먼저 가로채야 해요.

**`subscribeWithSelector`가 그 다음인 이유** — 상태 구독 기능은 store 구조와 불변성 처리가 끝난 뒤에 붙어야 합니다. immer가 만들어준 최종 상태를 감지해야 하기 때문이에요.

**`persist`가 subscribeWithSelector 바깥인 이유** — 구독까지 설정된 완성된 store를 스토리지에 저장해야 합니다. 안쪽에 있으면 아직 구독 기능이 붙기 전 상태를 저장하게 돼요.

**`devtools`가 가장 바깥인 이유** — 모든 미들웨어가 적용된 최종 store의 상태 변화를 DevTools로 보여줘야 하기 때문입니다. 가장 마지막에 전체를 감싸야 모든 액션이 기록돼요.

```
안쪽 → 바깥쪽
combine → immer → subscribeWithSelector → persist → devtools
[핵심]   [불변성]  [구독]                  [저장]    [디버깅]
```
