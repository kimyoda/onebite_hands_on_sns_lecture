## 3.3 Zustand_store_actions_customHook

### 1. Store 구조 더 이해하기

- set, get - store의 두 가지 핵심 도구
- `create` 함수는 `set`과 `get` 두 가지를 제공한다.

```tsx
export const useCountStore = create<Store>((set, get) => {
  ...
})
```

- `set`은 상태를 변경할 때 사용한다. 두 가지 방식으로 쓸 수 있다.

```ts
// 직접 값 전달
set({ count: count + 1 });

// 함수형 업데이트
set((store) => ({ count: store.count + 1 }));
```

- 함수형 업데이트는 항상 최신 값을 기반으로 업데이트하기 때문이다.
- 직접 값을 전달하는 방식은 쿨로저 문제로 오래된 값을 참조할 수 있다.

- `get`은 현재 상태를 읽을 때 사용한다.
- action 함수안에서 다른 상태값이 필요할 때 쓴다.

```ts
increase: () => {
  const count = get().count; // count 값을 읽는다
  set({ count: count + 1 });
};
```

### 2. actions 패턴 - 상태와 액션을 분리

- 상태와 액션이 같은 레벨에 섞여 있는다.

```ts
type Store = {
  count: number;
  increase: () => void;
  decrease: () => void;
};
```

- 위 처럼 하면 상태인지 함수인지 구분이 어렵다.
- `actions`라는 객체로 액션틀을 묶어서 분리하는 패턴을 쓴다.

```ts
type Store = {
  count: number;
  actions: {
    increase: () => void;
    decrease: () => void;
  };
};

export const useCountStore = create<Store>((set) => ({
  count: 0,
  actions: {
    increaseOne: () => set((store) => ({ count: store.count + 1 })),
    decreaseOne: () => set((store) => ({ count: store.count - 1 })),
  },
}));
```

- 상태가 어떤건지, 액션이 무엇인지 한눈에 확인 가능, selector 구독 최적화에도 편하다.

---

### 3. Selector - 필요한 것만 구독

- Zustand는 store 전체를 구독하는게 아닌 필요한 값만 선택해서 구독할 수 있다.
- selector 함수라 한다.

```ts
// store 전체 구독
const { count, increase, decrease } = useCountStore();

// selector를 count만 구독 - count가 바뀔 때 리렌더링
const count = useCountStore((store) => store.count);

// actions 객체 구독 - actions는 절대 바뀌지 않으니 리렌더링 없음
const { increaseOne, decreaseOne } = useCountStore((store) => store.actions);
```

- actions를 별도 객체로 분리, selector가 최적화가 자연스럽게 된다.
- 액션 함수는 상태가 바뀌어도 새로 생성되지 않아, actions만 구독하는 컴포넌트는 리렌더링이 발생하지 않는다.

---

### 4. Custom Hook - store 로직을 컴포넌트에 분리

- selector를 컴포넌트에 쓰는것보단, Custom Hook으로 한번 더 감싸는 패턴이 더 많이 쓰인다.

```ts
// store/count.ts 하단에 Custom Hook 정의

// count 상태만 꺼내는 hook
export const useCount = () => {
  return useCountStore((store) => store.count);
};

// increase 액션만 꺼내는 hook
export const useIncreaseCount = () => {
  return useCountStore((store) => store.actions.increaseOne);
};

// decrease 액션만 꺼내는 hook
export const useDecreaseCount = () => {
  return useCountStore((store) => store.actions.decreaseOne);
};
```

- 이를 통해 아래처럼 변화

```tsx
// Viewer.tsx — count만 필요
export default function Viewer() {
  const count = useCount(); // store 구조를 몰라도 됨
  return <div>{count}</div>;
}

// Controller.tsx — 액션만 필요
export default function Controller() {
  const increase = useIncreaseCount();
  const decrease = useDecreaseCount();
  return (
    <div>
      <Button onClick={decrease}>-</Button>
      <Button onClick={increase}>+</Button>
    </div>
  );
}

// CounterPage.tsx — 그냥 조합만
export default function CounterPage() {
  return (
    <div>
      <h1>Counter</h1>
      <Viewer /> {/* count만 구독 */}
      <Controller /> {/* actions만 구독 */}
    </div>
  );
}
```

- 관심사 분리: 컴포넌트는 UI, store는 상태만 담당
- 리팩토링 용이: store 구조가 바뀌어도 Custom Hook만 수정하면 된다.
- 자동 최적화: `Viewer`는 count가 바뀔 때, `Controller`는 actions를 구독, 리렌더링이 없다.
